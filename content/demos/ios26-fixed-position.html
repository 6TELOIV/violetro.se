Fixed Position Elements in iOS 26 Safari

The gradient background should stay fixed, while the rest of the page scrolls in front of it.

However, on iOS 26, the fixed position element can be detected as a modal, popup, sheet, or other UI element that the system decides needs to be inset below the browser chrome, and fills the browser chrome opaque.

I believe this goes against the intention of viewport-fit=cover, which opts websites in to display their content as full as possible, using safe-area-inset to ensure the important/non decorative content doesn't hide behind screen notches, browser chrome, system UI, etc.
